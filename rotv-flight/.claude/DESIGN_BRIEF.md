# ROTV Flight Product Overview

The new ROTV Flight product aims to redesign and enhance the Remote Operated Towed Vehicle (ROTV) control and data acquisition system, focusing on integration with EIVA hardware platforms, improved usability, safety, and autonomous operation. The project is a strategic initiative to unify and upgrade the software interface, ensuring it meets the evolving needs of diverse users operating in complex subsea environments, whilst maintaining the functionalities customers value today.

---

# Task

Design and deliver a user-centric ROTV control and monitoring software platform that ensures ease of use and safety in subsea operations. It should:

- Support autonomous and manual vehicle operation modes
- Provide clear, real-time system health and telemetry data
- Enable effective remote support and troubleshooting
- Integrate seamlessly with hardware and external data acquisition systems
- Prioritize features for a minimum viable product (MVP) while laying groundwork for future scalability

# Needs

In offshore operations, safety is paramount. Operators require:

- A clear, simple overview of ongoing tasks
- Ease of use and intuitive workflows
- Correct placement of the ROTV in the water
- Real-time system status updates
- Auto-detection of instruments to reduce setup errors

Effective systems must include:

- Runline information
- Data quality checks
- Assurance of no data gaps
- Emergency readiness
- Easy troubleshooting tools
- Historical records for insight, knowledge base, and better decision-making

# Features

## Must-haves

- Pre-flight check routines and system setup automation
- Real-time telemetry and system health monitoring

# Risks and Mitigations

## Hardware/software crashes

**Mitigation:** Robust error handling, automated checks, quick emergency stop options.

## Complex setup leading to user errors

**Mitigation:** Automated detection, simple preflight checks, guided tutorials.

## Data quality issues

**Mitigation:** Real-time QC tools and calibration support.

## Integration challenges

**Mitigation:** Modular APIs and thorough testing.

## User resistance due to learning curve

**Mitigation:** Prioritise usability, consistent UI, on-demand help materials.
